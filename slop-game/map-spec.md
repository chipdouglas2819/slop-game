# Slop City — final build spec (judge output)

## Chosen
Proposal 1 — "Slop City — a living skyline terrarium" as the spine, with grafts: P2's one-pip status discipline, deadpan zombie-caption ladder, one-time "tap the glowing thing." hint, permanent-litter joke, and CustomEvent card-focus mechanism; P3's bucketed particle counts, max-3-button bottom-sheet popover pattern, scandal-taps-route-to-the-existing-interrupt rule, staged ship order, and prefers-reduced-motion handling. P1's docked skyline strip, birds/drones/coins, and P3's view toggle are cut.

## Rationale
The owner's bar is watchable delight + one-glance status + ≤1 new interaction + visible satire, buildable in one session. Slop City wins on the two hardest criteria: watchable delight (buildings, lit windows, roof beacons blinking at the economy's tempo, smoke, sirens — an ant farm, where P2's constellation is dots-and-dashes and reads as a chart) and satire (growth and rot in ONE frame: your skyline rises while the sky goes toxic and the sidewalk crowd turns robotic — the thesis with zero copy). P3's full-screen orrery is the strongest genre answer but fails two hard constraints: a view toggle is a second navigation mode (new chrome, the IA-seam risk its own author flags) and ~750 lines is not one session. P2 fails none of the constraints but underdelivers the FUN mandate — its real contributions are discipline rules and jokes, which graft cleanly onto P1's body. P1's risks are all addressable by subtraction: cutting the IntersectionObserver strip, ambient fauna, and coin-drops brings it to ~620 lines while keeping every load-bearing tell. Both grafted proposals independently invented the same anti-management payload (one gold next-best-action glow + a pure selector), which confirms it as the core mechanic; P1's popover version is kept because it actually removes management taps (publish/buy/hire without scrolling seven cards), not just scanning — and it reuses the app's existing bottom-sheet language, so it costs no new UI vocabulary. Verified against code: zero engine changes needed; every selector the spec reads already exists.

Tapping the strip expands.

## nodeAnatomy
TIER from units via MILESTONES [25,50,100,200,300,400]: tier = units===0 ? 0 : 1 + (count of milestones ≤ units). TIER TABLE (body height px, lit-window grid cols×rows): t1 shack (18, 1×1), t2 shop (30, 2×2), t3 office (44, 2×3), t4 tower (58, 2×4) + 2px antenna line, t5 high-rise (72, 3×4) + neon sign rect (8×4 fill platform color, opacity .9), t6 skyscraper (86, 3×4) + satellite dish (circle r=3 + tick), t7 megatower (96, 3×4) + rooftop billboard: 10×8 rect + the page's current TOPIC_EMOJI as 7px SVG <text> (static; updates only on retune). Window = 5×4 rect, 3px gaps, from the bottom up; cap 12 windows/building (~70 window rects worst case, fine for static SVG). Building body fill #23232e, 1px stroke = COLORS[platform] at 60% opacity; ground apron rect (lot width × 4px at y=136) fill COLORS[platform] opacity .35; door = 5×7 rect at base center; platform EMOJI as 7px <text> sign beside the door. LIT STATES: manager hired → ALL windows lit (fill #fde68a, opacity .9, each with `winFlicker 7s infinite` at delay -i*1.3s — 'the building runs itself'); manual page → only window[0] lit; units 0 → none. STATUS OVERLAYS, exactly one roof pip per lot (P2's priority rule), evaluated: (1) SCANDAL (activeScandal.pageIdx === pageIdx): DOM pip '🚨' 12px above roof, opacity-pulse 0.9s loop, + SVG red wash rect over the building (fill #ef4444 opacity .12); (2) OVERUSED (saturationMult(state.saturation[recipeKey(page.recipe)]) < SATURATION_OVERUSED_BELOW, only when topicChipUnlocked): two 6px gray DOM puff spans above roof on `smokePuff 2.4s infinite` (delays 0/-1.2s) + window group opacity .45; (3) TAP-NEEDED (units>0 && !manager && cycleProgress===0): '👆' 12px pip on `fingerBob 1.2s infinite`. LOSING MONEY (manager && pageDollarsPerSec()<0): lit windows fill #f87171 + `redFlicker 1.4s infinite` (coexists with pip — it's a window state, not a pip). QUIET AFFORDABLE tier: door fill flips to steady #f59e0b (no pulse) when (next-milestone buy cost ≤ money) OR (!manager && managerCost ≤ money); else #1f1f28. CRACKDOWN (state.crackdown && platform matches && lastTickAt < untilMs): district-wide, coexists with pips — every lot of that platform gets ground apron `crackStrobe 1.2s infinite` cyan tint (#22d3ee), a '🚓' 10px span at curb, and ONE shared drain bar under the district: DOM div, 2px tall, cyan, `transform-origin:left`, inline `animation: sweepDrain Ns linear forwards` where N=(untilMs−Date.now())/1000 computed once at mount, keyed `key={crackdown.untilMs}` so it never restarts. LOCKED LOT: dashed rect 38×26 (stroke #3f3f46 dasharray 3 3, fill #101016), '🏗' 8px text, two-line 5.5px text 'ZONED FOR' / 'SLOP' fill #52525b rotate(-6°); when both unlock reqs met, stroke + text turn amber (#f59e0b) — and it becomes the beacon target via priority 2. THE GOLD BEACON (exactly one on the whole map, ever): DOM ring div around the target lot — left i*50.14+3, width 44, bottom 24, height TIER_H[tier]+16 (locked: 36), `rounded-xl border-2 border-amber-400` + an inner pre-blurred glow div (inset-0, bg amber-400/25, blur-[6px] set ONCE, not animated) — the pair animates `beaconPulse 1.6s infinite` (transform scale 1→1.05 + opacity, NO box-shadow animation). Amber is deliberately distinct from the app's fuchsia/red pulses. MILESTONE one-shot: building body <g> gets `key={tier}` and class `svg-grow animate-[buildingGrow_500ms_ease-out]` — remounts only when tier changes (utility CSS: .svg-grow { transform-box: fill-box; transform-origin: 50% 100%; }).

## liveness
All continuous motion is infinite CSS keyframes on stable-keyed nodes; React only swaps classes/props when a BUCKET changes. (1) PARTICLES — per live lot, absolutely-positioned DOM spans at lot center, bottom = 28+TIER_H+4: count = manual ? 1 : (units ≥ 200 ? 3 : 2); content = TOPIC_EMOJI[page.recipe.topic] at 10px (map defined in mapLogic.ts: shrimp_jesus 🦐, africa_boys 🖼, sob_bait 😢, mushroom_guide 🍄, fake_memoir 📖, lofi 🎧, finger_family 👶, glue_pizza 🍕, recipe_mill 🍲, agree_fable 💼); animation `slopRise` (translate(var(--drift), -86px) + opacity 0→.9→0), duration 3 + lotIdx*0.25 + j*0.7 s, delay -(j*1.3 + lotIdx*0.4) s, --drift = (j−1)*8px; stable keys `${lotIdx}-${j}` so loops never restart. Manual pages: the single particle's `animation-play-state` is `paused` unless cycleProgress > 0 (passed as boolean `inFlight` — NEVER the float) — tapping Publish makes the building literally exhale a post. (2) TEMPO BLINK — each producing MANAGED building has a 3px roof beacon rect with `roofBlink` at inline `animation-duration: max(0.5, effectiveCycleSec(slot, units))s` — SAFETY CLAMP REQUIRED: a maxed 1s-base page reaches ~15ms cycles; never let any loop run faster than 2Hz. Duration only changes at milestone halvings, so the inline style is stable between milestones; the restart on crossing reads as the level-up. Fast pages strobe (at 0.5s floor), slow ones breathe — the city's rhythm IS the economy. (3) WINDOW TWINKLE — shared `winFlicker 7s` with per-window negative delays (above). (4) CROWD — 8 spans on `crowdShuffle 6s ease-in-out infinite alternate` (translateX ±5px), delays -(i*0.8)s. (5) SMOKE/SIREN/FINGER/BEACON/STROBE/SWEEP per nodeAnatomy. (6) BLACKOUT one-shot (see zombieProgression). Minute 1: one shack, one lit window, dusky purple sky, one particle per Publish tap, 8 humans strolling. Hour 2: five towers blinking at different tempos, ~12 particles streaming, sky going green, half the crowd robotic, litter on the sidewalk — the delta is pure state, nothing scripted.

## zombieProgression
zQ = Math.round(zombieRatio(state)*20)/20 (5% steps; style objects memoized per step in a Map). SKY: per-channel hex lerp dusk→toxic: --sky-top lerp(#1a2240 → #0c1f16, zQ), --sky-horizon lerp(#4a3060 → #2f5a36, zQ), set as inline CSS vars on the wrapper; SVG gradient stops reference the vars. THE CORE: Era I sun --core = lerp(#fbbf24 → #86efac, zQ); Era II+ eye --core = lerp(#a78bfa → #4ade80, zQ) — 'the open internet' literally sickens as Z climbs, and every particle you built flies into it. CROWD = the ratio made flesh: robots = Math.round(zQ*8); spans render '🤖' for the first `robots` indices, '🚶' for the rest. At Z 80% one last human walks among machines; the towers blaze on regardless — the Dead Internet achievement literalized. LITTER (grafted from P2, free from the data model): count = lifetimeE ≤ 1e6 ? 0 : min(6, floor((log10(lifetimeE)−6)/2)+1); glyphs ['🗑','📰','🥡','📦','🥤','🗞'] in fixed positions. Because lifetimeE survives BOTH prestiges (verified in prestige.ts and the PULL_PLUG reducer), every reset collapses your buildings back to lots but the trash REMAINS — the world stays ruined, only your score resets. CAPTION LADDER (the deadpan teacher, grafted from P2/P3), picked by zQ: <0.10 'the internet (still alive)' · <0.30 'the internet (it's seen better days)' · <0.50 'the internet (mostly bots now)' · <0.80 'the internet (bots watching bots)' · ≥0.80 'the internet is mostly you now. pull the plug?'. PULL THE PLUG — the one cinematic the game earns: SlopCity watches state.lastEraJumpGain with a prev-value ref; on an observed null→number TRANSITION within a mounted session (never on mount — reload-safe), set local blackout=true for 2.6s and close any popover: lit windows get `blackoutWin` (opacity→0) at staggered delays lotIdx*0.12s, a full-bleed black overlay fades in 0.6s (delay 0.4s), a lone '▌' span blinks (`cursorBlink 1s steps(1)`) on the empty sidewalk, caption swaps to 'you turned it off. it needed that.' — then state clears and the city re-renders naturally as empty lots under a fresh z=0 dusk sky with humans back on the sidewalk (the new era's fresh audience — thematically correct, costs nothing).

## interactions
ONE new interaction concept, total: tap a thing on the map → a mini bottom-sheet remote control for surfaces that already exist. (a) TAP A LIVE LOT → MapPopover (same fixed inset-0 z-50 backdrop + rounded-t-2xl bottom-sheet language as TuneSheet, uses useLockBodyScroll, sfx('uiOpen'), backdrop-tap dismisses). Content: header row (platform avatar via exported COLORS/EMOJI, slot.name, ×units, right-aligned live rate — manager ? `fmtMoney(pageDollarsPerSec)/s` : `fmtMoney(tap net)/post`, red when negative) + AT MOST 3 buttons, all existing dispatches: [Publish] (units>0 && !manager; fuchsia, with the same cycleProgress width-fill as PublishButton; dispatch {type:'TAP'}; disabled in flight) · [Buy] (if nextMilestone m exists and unitCost(slot, units, m−units) ≤ money → label `Buy to ×{m} — {fmtMoney(cost)}`, dispatch BUY_UNITS count=m−units; else `Buy ×1 — {fmtMoney(cost)}`, disabled if unaffordable; units===0 → `Start this page — {cost || 'free'}` count=1; sfx('buy')) · third slot: no manager → [Hire Manager — {fmtMoney(managerCost)}] (dispatch BUY_MANAGER, disabled if unaffordable) / manager && topicChipUnlocked → [⚙ Tune] which mounts the existing <TuneSheet pageIdx> unchanged (local tuneOpen state) — the Tune sheet remains the only decision surface. Footer link `Details ↓` → onClose() then `window.dispatchEvent(new CustomEvent('slop:focus-page', { detail: { pageIdx } }))`. (b) TAP A LOCKED LOT → MapPopover locked mode: header `🔒 {slot.name} · {PLATFORMS[platform].name}`, slot.flavor line, the exact LockedSlotCard checklist (✓/• cash saved, ✓/• lifetime views), one button [Open it →] enabled iff both reqs met → dispatch UNLOCK_SLOT + sfx('unlock'). Read-only for not-next slots — UNLOCK_SLOT's own gates make this safe. (c) TAP THE SCANDAL BUILDING (activeScandal.pageIdx matches) → NO popover; `document.getElementById('scandal-interrupt')?.scrollIntoView({behavior:'smooth', block:'center'})` — the existing interrupt stays the resolution surface. (d) THE GOLD BEACON: pure selector `nextBestAction(state)` in mapLogic.ts returns at most ONE target, fixed priority: (1) activeScandal → {kind:'scandal', pageIdx} · (2) first locked PAGE_SLOTS entry with money ≥ unlock.cash AND lifetimeE ≥ unlock.lifetimeE → {kind:'unlock', slotId} · (3) first page with units>0 && !manager && money ≥ managerCost → {kind:'manager', pageIdx} · (4) first page where the buy-to-next-milestone bundle (or first unit when units===0) is affordable → {kind:'buy', pageIdx, count} · (5) topicChipUnlocked && first page units>0 with saturationMult < SATURATION_OVERUSED_BELOW → {kind:'retune', pageIdx} · else null. Exactly that lot wears the amber beacon ring. Tapping it opens the popover with the recommended button pre-glowed (`ring-2 ring-amber-400`, steady — one confirming tap, no accidental spends); kind 'retune' pre-glows [⚙ Tune]; kind 'scandal' routes per (c). First time a beacon ever renders, the caption shows 'tap the glowing thing.' until the first beacon tap (localStorage `slop.city.beaconHintDone`), then the ladder resumes forever. (e) CHEVRON collapse/expand (standard UI affordance, not a game concept). PageCard.tsx gains the ~14-line 'slop:focus-page' listener: matching pageIdx → setExpanded(true), rootRef.scrollIntoView({behavior:'smooth', block:'center'}), transient `ring-2 ring-amber-400` for 1.4s. GLOW EXCLUSIVITY RULE (hold the line): amber/gold appears ONLY on the beacon ring, the pre-glowed popover button, the focus flash, and quiet door lights — never pulsing anywhere else; fuchsia/red pulses remain the cards' and header's property.

## perfNotes
The tree already re-renders at 10Hz through one context, so the budget is prop-diffing, not subscriptions. Rules: (1) Building is React.memo with primitives-only props — verify once with React Profiler that steady-state ticks are 7 memo bails; (2) cycleProgress NEVER enters the map as a number — tempo lives in inline animation-duration (changes only at milestone halvings) and a paused/running class from the boolean inFlight; (3) zQ quantized to 5% steps, skyVars/caption/crowd memoized per step; (4) every keyframe animates transform/opacity only — no box-shadow, no layout properties, no rAF, no canvas; beacon glow is a pre-blurred static element whose opacity/scale animates; (5) particle/pip spans use stable keys so reconciliation never restarts loops; sweep bar keyed on crackdown.untilMs, duration computed once at mount; (6) animated-node ceiling ~40 (≤21 particles realistic-max ~12, 7 roof blinks, 8 crowd, ≤4 smoke, 1 beacon pair, 1 sweep) and ~120 static SVG rects — under the existing cards' per-tick width-style churn; (7) SAFETY: clamp every loop ≥0.5s (roofBlink floor — effectiveCycleSec reaches ~15ms on maxed early pages; a 60Hz strobe is a photosensitivity hazard and the clamp is mandatory); (8) emoji are accents only — structure stays geometric, so cross-OS glyph variance can never block comprehension; spot-check the ~14 glyphs used on one Android + one iOS device; (9) prefers-reduced-motion disables all decorative loops, status stays color-static.
